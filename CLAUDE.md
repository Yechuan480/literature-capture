# Claude instructions for literature-capture

## Git workflow

When you finish meaningful changes in this project, **automatically**:

1. Stage relevant files (never secrets, PDFs, `_captures/`, `data/`, `.venv/`)
2. Create a commit with a clear message
3. Push to `origin` (`git@github.com:Yechuan480/literature-capture.git`, branch `main`)

Do this without waiting for the user to say “commit” or “push”.  
Do not force-push or amend published commits unless explicitly asked.
